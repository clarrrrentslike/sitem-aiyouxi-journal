# sitem-aiyouxi-journal

本仓库用于归档和发布多个独立的 HTML 页面，内容涵盖信息展示、工具页面、静态文档等常见网页形式。仓库不针对任何特定域名或网站，仅作为页面资料存储与分发的基础载体。

## 项目简介

这是一个以 HTML 文件为核心的静态资源仓库，主要用于存放零散的、独立运行的网页文件。每个页面均为自包含的 HTML 文档，可直接在浏览器中打开，无需后端服务支持。

仓库的用途包括但不限于：
- 保存与展示信息类页面
- 提供简单工具或交互界面
- 归档历史静态文档
- 作为个人页面资料的备份

## 目录结构

```
sitem-aiyouxi-journal/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的资源文件（如图片、样式、脚本）
├── archive/        # 旧版本或不再维护的页面存档
└── README.md       # 本文件
```

- **pages/**：仓库的主体目录，每个子文件夹或文件对应一个独立页面。
- **assets/**：为页面提供支持的静态资源，按需组织。
- **archive/**：存放已被替换或不再更新的页面，保留历史版本。

## 页面归档说明

仓库中的页面遵循以下基本规范：

1. **独立性**：每个 HTML 文件应尽量不依赖外部资源（除 assets 目录外），确保离线可预览。
2. **命名**：页面文件以有意义的英文或拼音命名，避免使用特殊字符。
3. **版本**：若页面有更新，旧版本移至 `archive/` 目录，并保留原文件名加时间戳后缀。
4. **引用**：页面内引用的资源路径使用相对路径，指向 `assets/` 目录。
